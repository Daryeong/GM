Project 02

Popup Screen Study

기본 팝업, 선택형 팝업, 보상형 팝업, 리스트형, 상세정보형 화면을 모아 상황에 따라 어떤 정보와 버튼이 먼저 보여야 하는지 정리했습니다.

Goal

팝업의 목적에 맞게 안내 문구와 버튼 구성을 정리하기

Process

팝업 유형 모으기 → 레이아웃 비교 → 버튼 순서 정리 → 문서로 정리

Result

확인, 선택, 보상, 상세정보 팝업을 만들 때 참고할 기준을 정리했습니다.

Tools

UI 분석 · 화면 구성 · PDF 문서 정리

기본 팝업 화면 확인 또는 안내 목적의 기본 팝업으로, 핵심 메시지와 단일 CTA가 먼저 보이도록 구성했습니다. 소요 시간: 1일 선택형 팝업 화면 사용자가 두 가지 이상 선택지를 비교하고 결정할 수 있도록 버튼 위계와 문장 길이를 정리했습니다. 소요 시간: 1일 보상형 팝업 화면 획득 보상과 다음 행동을 분리해 보상 인지와 후속 행동을 자연스럽게 연결했습니다. 소요 시간: 1일 리스트형 팝업 화면 복수 항목을 스캔하기 쉽도록 목록 구조, 간격, 항목 구분 방식을 중심으로 정리했습니다. 소요 시간: 1일 상세정보 팝업 화면 세부 정보 확인이 필요한 상황을 고려해 제목, 본문, 보조 정보의 읽는 순서를 설계했습니다. 소요 시간: 1일

Working Process

작업을 정리하는 방식

  1. 살펴보기비슷한 화면과 사용 흐름을 먼저 살펴봅니다.
  2. 정리하기화면에 꼭 필요한 정보와 버튼을 정합니다.
  3. 만들기프레임, 버튼, 게이지, 팝업을 실제 리소스로 제작합니다.
  4. 확인하기정보가 잘 보이는지, 클릭해야 할 요소가 분명한지 확인합니다.
