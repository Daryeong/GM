Working Process

Working Process

  1. Observe비슷한 화면과 사용 흐름을 먼저 살펴봅니다.
  2. Organize화면에 꼭 필요한 정보와 버튼을 정합니다.
  3. Create프레임, 버튼, 게이지, 팝업을 실제 리소스로 제작합니다.
  4. Review정보가 잘 보이는지, 클릭해야 할 요소가 분명한지 확인합니다.

Photoshop UI Study

Photoshop UI Reference Study

기본 팝업, 선택형 팝업, 보상형 팝업, 리스트형, 상세정보형 화면을 모아 상황에 따라 어떤 정보와 버튼이 먼저 보여야 하는지 정리했습니다.

Goal

팝업의 목적에 맞게 안내 문구와 버튼 구성을 정리하기

Process

팝업 유형 모으기 → 레이아웃 비교 → 버튼 순서 정리 → 문서로 정리

Result

확인, 선택, 보상, 상세정보 팝업을 만들 때 참고할 기준을 정리했습니다.

Tools

UI 분석 · 화면 구성 · PDF 문서 정리

기본 팝업 화면확인 또는 안내 목적의 기본 팝업으로, 핵심 메시지와 단일 CTA가 먼저 보이도록 구성했습니다.소요 시간: 1일선택형 팝업 화면사용자가 두 가지 이상 선택지를 비교하고 결정할 수 있도록 버튼 위계와 문장 길이를 정리했습니다.소요 시간: 1일보상형 팝업 화면획득 보상과 다음 행동을 분리해 보상 인지와 후속 행동을 자연스럽게 연결했습니다.소요 시간: 1일리스트형 팝업 화면복수 항목을 스캔하기 쉽도록 목록 구조, 간격, 항목 구분 방식을 중심으로 정리했습니다.소요 시간: 1일상세정보 팝업 화면세부 정보 확인이 필요한 상황을 고려해 제목, 본문, 보조 정보의 읽는 순서를 설계했습니다.소요 시간: 1일

Photoshop Original UI Design

UI Resource Design

RPG 게임에서 자주 쓰이는 정보 패널, 진행도, 캐릭터 슬롯, 액션 버튼을 제작한 작업입니다. 플레이어가 현재 상태를 빠르게 보고 다음 행동을 고를 수 있도록 구성했습니다.

Goal

게임 안의 상태 정보와 주요 버튼을 한눈에 알아볼 수 있게 만들기

Process

화면 요소 분류 → 중요한 정보 정리 → 스타일 맞추기 → 리소스 제작

Result

출석, 웨이브, 캐릭터 관리, 던전 입장 화면에 쓸 수 있는 UI 리소스를 만들었습니다.
